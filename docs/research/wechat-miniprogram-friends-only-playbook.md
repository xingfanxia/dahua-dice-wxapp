# 微信小程序 · 朋友间私用（非商用）可行性与做法

> Researched 2026-06-11 via multi-agent web research; compliance-critical claims adversarially verified against live official docs (developers.weixin.qq.com / mp.weixin.qq.com / kf.qq.com) the same day. 政策性内容时效敏感 — 引用前留意日期，重大决策前 spot-check 官方页面。
>
> 适用场景：个人开发者想做一个**只给朋友玩**的小程序（联机游戏、小工具、群内应用），不商用、不公开上架。结论先行：**完全可行，且零备案、零审核、零版号、零认证 —— 关键是永远停留在体验版。**

## 1. 核心机制：备案/版号的触发点是"上架"，不是"商用"

官方原话（微信官方社区回复，2024，并于 2026-06-11 对官方备案 FAQ 实时核验仍在）：

> "未备案的小程序仅允许使用「开发版」和「体验版」；必须完成备案后才可提交代码审核及正式版本发布。"

- ICP 备案（2023-09-01 起强制）、代码审核、微信认证、游戏备案/版号 —— **全部只在"提审/上架"这一步触发**。未备案上架报错 `errcode 86369`。
- 体验版没有过期时间（微信的体验版构建不自动失效；"7 天二维码"是**支付宝**小程序的规则，勿混淆）。截至 2026-06 未发现任何收紧体验版使用的政策。
- 因此"内部朋友用"的正确姿势：注册 → 开发 → 上传体验版 → 加体验成员 → **永远不点提审**。
- 如果哪天想公开上架（游戏类内容）：对个人开发者是双轨制 —— 带内购（IAP）需要完整**游戏版号**（个人法律上拿不到）；免费/广告变现（IAA）免版号但需**小游戏备案**（省级出版主管部门前置审批，官方 10-20 工作日、个人实测 30-40+）+ ICP 备案。个人主体 2026 年仍可发布小游戏，但被明文排除在**牌类/角色类/文化互动**三个类目之外；且**牌类专项规则**另行禁止私密房间口令、自定义头像昵称、胜者排行榜、带比分的分享图——这些恰好是房间制骰子游戏的核心功能。把游戏内容挂在非游戏类目（工具等）属于明文违规的"类目逃避"，提审必被打回。现存的大话骰小游戏（如哈局大话骰）均为企业主体运营。结论：个人公开上架骰子游戏 ≈ 不可行；想公开需要企业主体或发行方合作。

## 2. 人数上限（真正的约束）

个人主体每个小程序（官方 kf.qq.com 数据，高置信）：

| 角色 | 上限 | 说明 |
|---|---|---|
| 体验成员 | **15** | mp.weixin.qq.com → 成员管理，按微信号逐个添加（对方需开启"可通过微信号搜索"）；或朋友扫体验码点"申请体验"再批准 |
| 项目成员 | **15** | 全部自带"打开体验版"权限，可计入玩家池 |
| 管理员 | 1 | |

≈ **31 人/appid**。个人可注册 **5 个**小程序，同一套代码部署多个 appid 可扩到 ~150 人（笨但可行）。每个微信号最多当 50 个小程序的体验成员。对比：未认证非个人主体 30/30；认证企业 90/90。

## 3. 要"申请"的东西（全流程）

1. **注册个人主体小程序**（mp.weixin.qq.com）：免费、当天完成。需要大陆身份证 + 实名认证的微信号当管理员（18+，一个身份证最多注册 5 个个人小程序）。类目选个"工具"（注意：游戏内容挂工具类目严格说是明文违规的"类目逃避"——体验版路线之所以没事，纯粹因为永不提审、没人审查；一旦提审必被打回）。注册成普通**小程序**，不要注册小游戏账号（小游戏账号会触发游戏资质门槛，且个人主体牌类/角色类类目已关闭 —— 2026-06-11 对官方类目页实时核验仍然如此）。
2. **开通云开发**（开发者工具里一键）：2025-02-19 起每个小程序账号送一个**免费云开发环境，条件恰好是"未发布上线"**（活动现写到 2026-12-31）。发布后 15 天转收费 —— 朋友局永不发布，正好白嫖。
3. **加体验成员**：一次性手工操作。
4. 没了。无备案、无认证、无版号。
   - 认证的精确说法：2023 年起新的**小程序认证**确实覆盖个人主体（30 元/次，年审制；旧文档"个人不能微信认证"说的是旧版微信认证，勿混淆）。未认证的后果是限制**发布新版本/搜索/分享等上架后能力** —— 全部作用于发布链路，体验版路线不触发，可以不交这 30 元。

身份硬性要求：个人主体注册 + 备案体系实质要求大陆身份证与可收短信的 +86 手机号；人在海外但持大陆身份证可全程远程完成。无大陆身份证则无法注册个人主体小程序。

## 4. 入口与社交能力（体验版内全部可用）

进入路径：扫体验版二维码（版本管理页生成，可配 path+query）→ 之后常驻"最近使用"列表；也可用服务端 API `getunlimitedqrcode`（`env_version: "trial"`, `check_path: false`）生成**永久有效**的体验版小程序码。

体验成员之间的社交闭环（均为个人主体可用、体验版内有效）：

- **分享卡片带参数**：`Page.onShareAppMessage` / `button open-type="share"`，自定义 title / imageUrl(5:4) / path（房间码塞 path）→ 单聊群聊均可；接收方若也是体验成员，点开直达体验版对应页面；非成员点开报无权限（可从二维码申请体验）。
- **动态消息**（updatable message）：分享卡片可实时更新状态（"X/Y 人已加入"）—— 为游戏房间邀请量身定做。
- **群识别**：`shareTicket` + `wx.getGroupEnterInfo`（取代已废弃的 `wx.getShareInfo`）拿 opengid，判断同群。
- **登录**：`wx.login` → `code2Session` 拿 openid，静默匿名身份。UnionID 个人基本拿不到（需付费开放平台认证）——单 appid 场景用不上。
- **头像昵称**：`getUserProfile` 自 2022-10-25 起只返回"微信用户"+灰头像；用**头像昵称填写能力**（`<input type="nickname">` + chooseAvatar button）。
- **订阅消息**：一次性模板可用（一次点击=一条推送，可做"游戏开始了"通知）；长期模板仅公共服务类，游戏/工具拿不到。
- ❌ **小游戏专属**（普通小程序没有）：好友排行榜/开放数据域、`wx.shareMessageToFriend` 定向分享、程序化主动 `wx.shareAppMessage`。
- ❌ **web-view 对个人主体封禁** —— "套壳现有 web 应用"这条路不存在。

## 5. 后端：云开发（CloudBase）是个人开发者的默认答案

- `wx.cloud.callFunction` / 数据库调用**完全绕过 request 合法域名白名单**（合法域名必须 HTTPS/wss + ICP 备案域名，所以 `*.vercel.app`、Upstash 等海外后端从小程序根本调不到）。
- **实时同步**：云数据库 `db.watch`（基础库 2.8.1+，where/orderBy/limit，≤5000 docs/watcher，自动重连）—— 官方明确推荐用于回合制/棋牌状态同步。腾讯自家联机引擎 MGOBE 已于 2022-06-01 关停，别再找它。无延迟 SLA，社区经验亚秒到数秒、偶发掉监听 —— **生产质量需要 watch + 轮询兜底双通道**。
- **原子性**：云函数内单文档事务（~30s 上限）+ 条件更新 CAS（`where({version}).update(...)` 检查 `stats.updated`）可复刻 Redis Lua 的 version-CAS 模式；多文档强事务弱，状态尽量收敛到单 room 文档。
- **配额注意**（数据点冲突，建当时核验）：免费/基础套餐时代环境级并发 watch 连接很宽裕（官方案例数千并发监听）；但 2026 资源点计费的"个人版 ¥19.9/月"被查到**并发实时推送连接仅 10 条**（每客户端一个 watch = 1 条），标准版 ¥199/月=500 条。计费模型近年反复改版（按量→套餐→资源点），动手前以当时官方价目页为准。
- 免费环境额度（2025 活动）：20 万次调用/月（读写+函数）、2GB 存储、500 QPS —— 几十人的朋友局绰绰有余。

## 6. Web 技术栈 → 小程序的移植要点

- **Taro 4.x** 是唯一现实的 React 复用路径，但 **钉死 React 18**（react-reconciler 0.29，React 19 支持截至 2026-02 仍是 open issue）；Tailwind 走 weapp-tailwindcss v4（2025-03 发布）。
- **WXSS 不支持 oklch / color-mix / @layer** → postcss-preset-env 编译降级；keyframes/transform/opacity/CSS 自定义属性都支持，DOM+CSS 动画大体能活，但所有元素要换 View/Text 原语（无 document/window）。
- **没有 EventSource/SSE**：选 db.watch（推荐）或 `wx.connectSocket`（wss + 备案域名）或 `wx.request` 轮询。
- **音频**：Howler.js 跑不起来；`wx.createWebAudioContext`（基础库 2.19.0+）可做精灵图切片，或 InnerAudioContext。
- **传感器**：`wx.onDeviceMotionChange` / `onAccelerometerChange` 无权限弹窗（比 iOS Safari 省事）；震动 `wx.vibrateShort`。
- 纯 TS 业务逻辑（游戏引擎、校验）连同单测可原样进云函数 —— 移植时最先抢救的资产。

## 7. 不走小程序的备选：H5 直接发群里

- 微信聊天内**可以打开未备案的海外 HTTPS 链接**（可能有"非官方网页"点击确认；朋友圈高频分享未备案域名会被限流；被投诉可被拦截）。私密朋友群点链接这条路今天**实际上是通的**。
- 但要知道形式上的定性：**H5 游戏被《微信外部链接内容管理规范》§2.5 明文列为禁止传播的外链内容** —— 执法是投诉/内容驱动的，小范围朋友群从未见被动出手的案例，但这条路在书面规则层面是"违规被容忍"而非"合规"，规模化/被投诉即死。
- 但 **`*.vercel.app` 在大陆已死**（DNS 污染 + 443 SNI 阻断，Vercel 官方 KB 2025-11 确认）→ 必须绑自定义域名（官方 `cname-china.vercel-dns.com`），无需备案但延迟高、可达性 best-effort。
- H5 拿不到微信社交 API（分享卡片带参/openid/群识别），"邀请好友"只能靠裸链接 —— 这是小程序路线的核心增量。

## 8. 运营红线（体验版也别碰）

1. **任何真钱/筹码兑换/红包元素零容忍** —— 赌博是腾讯对未上架内容唯一会主动出手的类别（灰产惯用体验版分发，风控盯着这个模式）。免费骰子/棋牌玩法本身无害，命名也别往赌上靠。
2. **账号闲置冻结**：mp 后台长期不登录会被系统冻结（可凭身份找回）—— 每隔几个月登录一次。
3. 体验版带"体验版"角标、成员可开调试菜单 —— 朋友局无所谓。
